## Overview

ctx is local-first for normal desktop use, but different kinds of data flow through different systems.

Use **Settings** -> **Analytics** and related preferences to manage product telemetry behavior where those controls are available.

## What to expect

- Your repository and task state stay on the machine or remote host you control.
- Model prompts and responses still go to whichever provider or router you connect.
- Product telemetry settings are separate from model/provider traffic.

## Work records and raw detail

Work records, command observations, artifacts, logs, summaries, and the local
search index live under the local ctx data root unless you explicitly export or
copy them. Search, context, report, timeline, and desktop Work Report responses
use redacted renderable/searchable data by default and do not include raw
transcript payloads.

Provider/model traffic is separate from Work record storage. It only happens
through the harnesses or provider-backed features you connect. Raw local
expansion or full-local export should be treated as trusted-machine-only data.

## Practical privacy operations

- Keep telemetry settings explicit and reviewed.
- Avoid sharing sensitive repository data in issue reports.
- Use minimal reproduction details when reporting bugs externally.
- Re-check analytics preferences after major updates or when moving between environments.

## Support-sharing guideline

When reporting issues, share:

- clear reproduction steps
- minimal relevant logs
- no unnecessary secrets or proprietary data
